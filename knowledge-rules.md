# 人机知识库规则

> 本文档定义了"人机知识"触发机制的工作方式，便于跨 agent 共享。

---

## 触发条件

当用户提到以下关键词时，触发知识文档生成：

- "人机知识"
- "生成人机知识"
- 或类似表述

---

## 执行动作

触发后，agent 执行以下操作：

1. **整理内容**：将之前的回答内容整理成结构化的知识文档
2. **生成双格式**：
   - `.md` (Markdown) — 适合 agent 间共享、版本控制
   - `.html` (HTML) — 适合网页发布、直接浏览
3. **存入目录**：`/home/damon/.openclaw/workspace/human_AI_knowledge/`
4. **命名规则**：根据内容主题命名，如 `a2a-protocol.md`
5. **更新索引**：同步更新 `index.html`，添加新文档卡片
6. **更新 README**：同步更新 `README.md` 的文档索引表格
7. **提交推送**：Git commit 并 push 到远程仓库

---

```
human_AI_knowledge/

```

---

## 文档规范

### Markdown 格式要求

```markdown
# 标题

> 元信息：来源、日期、版本等

---

## 概述
简要说明主题

---

## 核心内容
...

---

## 相关资源
链接、参考资料

---

## 一句话总结
便于快速理解
```

### HTML 格式要求

- 完整的 HTML5 文档
- 内嵌 CSS 样式，无外部依赖
- 响应式设计，适配移动端
- 美观的排版和配色

---

## 设计目的

1. **跨 agent 共享**：Damon 的多个 AI 伙伴（Outis、Nemo 等）可共享同一知识库
2. **跨机器同步**：通过 git/云盘同步到不同机器
3. **公开发布**：HTML 文件可直接部署到网站，供他人查阅
4. **知识沉淀**：将问答内容转化为可复用的知识资产

---

## 适用场景

- 技术概念解释（如 A2A 协议）
- 工具使用指南（如 OpenClaw 功能）
- 故障排查经验
- 最佳实践总结
- 任何值得沉淀的知识

---

## 注意事项

- 文档内容应相对**简洁且完整**，避免过于冗长
- 保持 md 和 html 两种格式**内容同步**
- 文件命名使用英文小写 + 连字符，如 `a2a-protocol.md`
- 技术文档注明**来源和日期**，便于追溯
- **必须同步更新索引**：每次新增知识文档时，必须同时更新 `index.html` 和 `README.md`，否则网页访问者看不到新内容
