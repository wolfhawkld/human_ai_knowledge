Transformer vs Mamba：两大架构的核心机制对比

创建时间：2026-04-12 · 来源：Damon + Metis 对话整理
Transformer Mamba 自注意力 SSM KV Cache

核心对比：Transformer 通过自注意力机制实现全局信息关联，以空间换取精确回溯能力；Mamba 通过选择性状态空间模型实现时序压缩，以"智能压缩"换取无限序列长度。本文深入剖析两者的核心机制原理。

Transformer：自注意力与 KV Cache

核心思想

Transformer 的核心是 自注意力机制：让每个 token 都能"看到"所有历史 token，计算它们之间的关联程度。

QKV 机制详解

每个 token 会生成三个向量：

向量作用直观理解
Q (Query)发出查询"我要找什么信息？"
K (Key)提供索引"我是什么内容，有什么标签？"
V (Value)提供内容"我的实际信息是什么？"

数学表达

Attention(Q, K, V) = softmax(Q·Kᵀ / √d) · V

运作流程（以"今天 天气 很好"为例）

Step 1: 每个 token 生成 Q, K, V
  "今天" → Q₁, K₁, V₁
  "天气" → Q₂, K₂, V₂
  "很好" → Q₃, K₃, V₃

Step 2: 计算注意力权重（每个 token 的 Q 去问所有 token 的 K）
  
  Token "天气" 的视角:
    Q₂ · K₁ = "天气" 对 "今天" 的关注程度 → 权重 w₁
    Q₂ · K₂ = "天气" 对自己的关注程度 → 权重 w₂
    Q₂ · K₃ = "天气" 对 "很好" 的关注程度 → 权重 w₃
  
  softmax 后: w₁=0.2, w₂=0.3, w₃=0.5

Step 3: 加权聚合（用权重取所有 V 的加权和）
  context₂ = w₁·V₁ + w₂·V₂ + w₃·V₃
  这就是 "天气" 在当前上下文中的"理解后的表示"

Step 4: 预测下一个 token
  context₂ + 原始 token 表示 → 前馈网络 → 预测
关键洞察：Q 问的是"相关性"，不是"答案"。Q 不是去找"最高的下一个 token"，而是在问"你和我有什么关系？我该多关注你？"。最终通过加权聚合所有 V，得到"上下文理解后的表示"，然后基于这个表示去预测下一个 token。

KV Cache：空间换精确

推理时，生成每个新 token 都需要计算它与所有历史 token 的注意力，这需要所有历史 token 的 K 和 V。如果不缓存，每次都要重新计算 → 效率极低。

缓存内容

KV Cache: [K₁, V₁, K₂, V₂, K₃, V₃, K₄, V₄, ...]
每个 token 的 K 和 V 都被保存
（不存 Q，因为 Q 只用于当前 token 的查询）

为什么"巨大"？

公式: KV_cache_size = 2 × layers × heads × d_head × seq_len

举例 Llama-2-70B:
  - 80 层 × 64 头 × 128 维 × 2 (K和V) × seq_len
  - 每个 token 约 1.3 MB
  - 100K token ≈ 130 GB 显存！

Transformer 的代价与收益

方面代价收益
训练高计算量 O(n²)完全并行，GPU 利用率高
推理KV cache 内存爆炸可精确回溯任意历史位置
长序列受限于显存全局视野，信息无损

本质：空间换精确回溯能力

Mamba：选择性状态空间模型

核心思想

Mamba 的核心是 选择性状态空间模型 (SSM)：用一个固定大小的状态向量 h 来"压缩"整个历史序列，通过逐 token 更新状态来传递信息。

SSM 基础原理

状态空间模型来自控制论/信号处理：

连续形式:
  h'(t) = A·h(t) + B·x(t)    # 状态如何演化
  y(t)  = C·h(t) + D·x(t)    # 如何从状态输出

离散化后（用于神经网络）:
  h_t = Ā·h_{t-1} + B̄·x_t    # 状态更新（线性递推）
  y_t = C·h_t                 # 输出

直观理解：RNN 的进化

传统 RNN:  h(t) = tanh(W·h(t-1) + U·x(t))
          问题：梯度消失，无法学习长距离依赖

LSTM:     h(t) = f·h(t-1) + i·x(t)   # f=遗忘门, i=输入门
          解决了梯度问题，但门控参数固定

Mamba:    h(t) = A(x)·h(t-1) + B(x)·x(t)
          关键创新：A 和 B 由当前输入动态生成！

选择性机制：Mamba 的核心创新

传统 SSM：A、B、C 是固定参数，对所有输入一视同仁。

Mamba：让 A、B、C 依赖于输入内容 → 模型学会"选择"什么该记住、什么该丢弃。

动态参数生成

def mamba_layer(x):
    # 动态生成 SSM 参数（选择性！）
    A = linear_A(x)  # 由输入决定"保留多少历史"
    B = linear_B(x)  # 由输入决定"新信息写多少进去"
    C = linear_C(x)  # 由输入决定"输出什么"
    
    # 扫描计算
    for t in range(seq_len):
        h = A[t] * h + B[t] * x[t]   # 状态更新
        y[t] = C[t] * h               # 输出
    
    return y

选择性参数的作用

参数作用直观理解
A(x)控制历史保留程度"旧状态还剩多少？"
B(x)控制新信息写入程度"新信息写多少进去？"
C(x)控制输出提取方式"从状态中取什么出来？"
核心洞察：模型学会了"什么重要、什么不重要"，实现了智能压缩。当遇到"25岁"时生成较大的 B 值写入状态；当遇到虚词时生成很小的 B 值丢弃噪声。

状态维度：固定大小的"记忆胶囊"

Transformer: 每个 token 存完整的 K, V (~KB per token)
             总量随序列线性增长

Mamba: 整个历史压缩到一个 h (~KB 总量，不增长!)

具体数字（Mamba-1.4B）:
  - d_state = 16（状态维度）
  - 每层状态 = 16 × 2048 × 4 bytes ≈ 128 KB
  - 24 层 ≈ 3 MB 总状态

对比:
  Transformer 100K token ≈ 130 GB
  Mamba 100K token ≈ 3 MB
  
差距：43000 倍！

Mamba 的代价与收益

方面代价收益
训练需硬件优化的并行扫描同样可并行训练
推理O(1) 固定内存极快，速度恒定
长序列信息可能有损压缩理论无限制，实测百万 token
回溯无法精确回溯历史细节可提取"压缩精华"

本质：压缩换无限长度能力

核心对比总结

Transformer

"我把所有历史都存着，你要什么我随时给你"

  • 精确，信息无损
  • 内存爆炸 (O(n))
  • 可精确回溯任意位置
  • 适合知识密集任务

Mamba

"我把历史压缩成精华，只给你最重要的"

  • 高效，固定内存 (O(1))
  • 信息可能有损
  • 无限序列长度
  • 适合长文档、边缘部署

计算复杂度对比

模型训练复杂度推理复杂度推理内存
TransformerO(n²)O(n) per tokenO(n) KV cache
MambaO(n) 并行扫描O(1) per tokenO(1) 固定状态

适用场景

场景推荐架构原因
知识密集型任务Transformer需精确回溯知识
RAG 系统Transformer需匹配检索结果
超长文档处理Mamba百万 token 不爆内存
流式生成Mamba恒定推理速度
边缘设备部署Mamba内存受限场景
需要精确引用Transformer可回溯原文位置

一句话总结

Transformer：空间换精确回溯能力（训练并行 + 推理可查任意位置）

Mamba：压缩换无限长度能力（固定内存 + 恒定推理速度）
