从老虎机比喻到 RAG 应用

核心概念

LinUCB (Linear Upper Confidence Bound) 是一种 Contextual Bandit 算法，将多臂老虎机问题引入上下文信息，广泛应用于推荐系统、RAG 精排等场景。

一、多臂老虎机比喻

赌场里有 K 台老虎机（K 个臂），每台有未知的"中奖概率"。目标是在有限投币次数内，最大化总收益。

困境：

  • 只玩已知收益高的机器 → 可能错过更好的机器
  • 到处尝试新机器 → 浪费在差机器上

这就是经典的探索-利用困境 (Exploration-Exploitation Dilemma)。

二、比喻 → LinUCB 对应关系

老虎机概念 LinUCB 对应 说明
老虎机（臂） chunk（候选文档） 每个 chunk 是一个"臂"
拉臂 选择返回某个 chunk 用户查询时选择一个 chunk
中奖概率 chunk 对该查询的匹配度 不是固定值，依赖查询
收益 用户反馈 reward 用户满意=高收益
投币次数 查询次数 每次查询是一次尝试
目标 最大化用户满意度 累积 reward 最大

三、传统老虎机 vs Contextual Bandit

传统老虎机（无上下文）

老虎机 A 的中奖概率 = 固定值 p_A（未知）

不管是谁拉、什么时间拉，中奖概率都一样。

问题：不考虑"谁在拉臂"。

Contextual Bandit（有上下文）

老虎机 A 的中奖概率 = 不固定！取决于"谁在拉"

例子
  • 老手拉老虎机 A → 中奖概率 0.9
  • 新手拉老虎机 A → 中奖概率 0.3

对应到 RAG：chunk_A 的"中奖概率" = 该 chunk 对当前查询的匹配度

  • 查询"信用卡申请" → chunk_A 匹配度高
  • 查询"信用卡还款" → chunk_A 匹配度低

四、LinUCB 公式详解

UCB 分数 = θᵀx + α × √(xᵀA⁻¹x)
部分 公式 含义
利用项 θᵀx 预测收益（点估计）
探索项 α × √(xᵀA⁻¹x) 不确定性 bonus

参数解释

θ（参数向量）：

θ = A⁻¹b

A = 累积矩阵 = I + Σ(xᵢxᵢᵀ)  (初始=单位阵)
b = 累积奖励 = Σ(rᵢxᵢ)      (初始=零向量)

θ 是线性回归的参数，通过历史数据学习得到。

xxᵀ（外积）

x 是向量，xᵀ 是其转置：

x = [x₁, x₂, ..., xₙ]  (行向量表示)
xᵀ = 列向量（转置）

xxᵀ = 外积 = 矩阵
例子（3 维）
x = [1, 2, 3]

xxᵀ = [1   2   3]
      [2   4   6]
      [3   6   9]

意义：xxᵀ 描述了向量 x 的"方向信息"，累积 xxᵀ 记录历史查询的分布。

xᵀA⁻¹x（二次型）

xᵀA⁻¹x = 向量 x 在"变形空间"中的长度平方

输出：一个数值（不是矩阵）
例子（2 维）
x = [1, 2]
A⁻¹ = [4  1]
      [1  3]

步骤 1: y = A⁻¹ × x = [6, 7]
步骤 2: xᵀ × y = 1×6 + 2×7 = 20

结果: xᵀA⁻¹x = 20

为什么表示不确定性：

  • 某个方向查询多 → A 在该方向值大 → A⁻¹ 在该方向值小 → 不确定性低
  • 某个方向查询少 → A 在该方向值小 → A⁻¹ 在该方向值大 → 不确定性高

α（探索参数）

α 越大 → 探索越多（更激进）
α 越小 → 利用越多（更保守）

默认 α = 1.0

五、完整流程

初始化

每个臂初始化：

  • A = I（单位阵）
  • b = 0（零向量）

选择动作

θ = A⁻¹b
点估计 = θᵀx
不确定性 = √(xᵀA⁻¹x)
UCB = θᵀx + α × √(xᵀA⁻¹x)
选择 UCB 最高

更新（收到反馈后）

A ← A + xxᵀ
b ← b + r × x

效果:
  - 正反馈 → b 增大 → θ 增大 → 未来更容易被选中
  - xxᵀ 累积 → 不确定性降低

六、为什么 LinUCB 有效？

上下文感知

不同查询，选择不同 chunk：

查询"信用卡申请" → θᵀx_A = 0.9 → 选 chunk A
查询"信用卡还款" → θᵀx_C = 0.85 → 选 chunk C

自动探索

新 chunk:
  A⁻¹ 不确定性高 → 探索bonus 高 → 会被尝试

尝试多次后:
  A 累积 → 不确定性降低 → 回归正常竞争

理论保证

LinUCB 的 regret 有界：

Regret = O(d × √(T × log(T)))

长期累积损失不会无限增长。

总结

要点 说明
核心 线性模型 + UCB 探索
