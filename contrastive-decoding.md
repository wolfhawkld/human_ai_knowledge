# Contrastive Decoding：用“强弱模型相减”凸显高质量答案

> 来源：Damon 与 Nemo 关于 contrastive decoding 的讨论  
> 日期：2026-05-01  
> 生成者：Damon + Nemo  
> 主题：LLM 解码策略、强弱模型对比、概率差分

---

## 概述

Contrastive decoding 是一种利用“强模型”和“弱模型”之间的概率差异来选择 token 或答案的解码方法。

它的核心不是证明“正确答案一定最大”，而是通过相减把弱模型也容易给高分的浅层模式、模板化答案、空泛答案压低，让强模型相对弱模型更有优势的候选凸显出来。

---

## 基本公式

一个常见的打分形式是：

```text
score(x) = log p_expert(x | c) - α · log p_amateur(x | c)
```

其中：

- `p_expert`：强模型对候选 `x` 的概率
- `p_amateur`：弱模型对候选 `x` 的概率
- `c`：当前上下文
- `α`：控制弱模型惩罚强度的系数

如果 `α = 1`，它近似于：

```text
log p_expert(x | c) - log p_amateur(x | c)
= log [p_expert(x | c) / p_amateur(x | c)]
```

也就是看“强模型相对弱模型，对这个候选多支持多少”。

---

## 直觉解释

Contrastive decoding 实际上在问：

> 哪个候选最能体现强模型相对于弱模型的能力增量？

它并不是单纯选择强模型概率最高的候选，而是偏好满足下面条件的候选：

```text
强模型支持度高，并且弱模型支持度没那么高
```

这背后的假设是：

- 弱模型容易高估常见、模板化、表面合理但不一定正确的续写；
- 强模型更可能识别出具体、正确、需要知识或推理能力的答案；
- 因此，把弱模型概率扣掉，可以削弱浅层统计模式，把强模型真正“多出来”的能力凸显出来。

---

## 它不能保证正确答案一定最大

Contrastive decoding 没有数学保证说：正确答案一定会在相减后排第一。

它依赖的是一个经验假设：

> 正确答案在强模型那里概率高，而在弱模型那里没有同样高；错误、空泛、模板化答案则更容易被弱模型也给出高概率。

当这个假设成立时，contrastive decoding 会很有效；当假设不成立时，它也可能失败。

---

## 常见失败情况

### 1. 弱模型也知道正确答案

如果某个问题很简单，弱模型也能准确回答，那么正确答案在弱模型那里也会有很高概率。相减后，正确答案的优势反而可能被削弱。

### 2. 强模型本身也错了

如果强模型对错误答案也很确信，那么 contrastive decoding 不能凭空纠正强模型的知识错误或推理错误。

### 3. 弱模型对某个奇怪候选概率极低

某些不合理 token 可能因为弱模型概率极低而获得很大的差值。为了避免这种情况，实际方法通常会加上 plausibility constraint，只在强模型认为还算合理的候选集合中做对比。

例如：

```text
V_head = { x : p_expert(x | c) >= β · max p_expert(· | c) }
```

意思是：先过滤掉强模型自己都不太相信的候选，再做强弱模型相减。

---

## 为什么它有用

很多 LLM 的生成问题不只是“不知道正确答案”，而是 decoding 时容易被高频、顺滑、模板化的模式吸走。

Contrastive decoding 的价值就在于：

- 保留强模型认为合理的候选；
- 惩罚弱模型也觉得很自然的浅层答案；
- 放大强模型相对于弱模型的知识、语义和推理优势。

因此它更像一种“浅层模式扣除器”，而不是真理筛选器。

---

## 与普通解码的区别

| 方法 | 主要选择依据 | 风险 |
|------|--------------|------|
| Greedy decoding | 每一步选择强模型概率最高的 token | 容易走向局部最优、模板化 |
| Sampling | 按概率采样，引入随机性 | 可能不稳定、产生低质量输出 |
| Beam search | 保留多个高概率路径 | 容易偏向通顺但保守的答案 |
| Contrastive decoding | 选择强模型相对弱模型优势最大的候选 | 依赖强弱模型差异假设 |

---

## 一句话总结

Contrastive decoding 不能保证正确答案一定胜出；它是通过“强模型概率 − 弱模型概率”来尽可能扣除浅层、常见、模板化模式，让更能体现强模型能力增量的答案凸显出来。
