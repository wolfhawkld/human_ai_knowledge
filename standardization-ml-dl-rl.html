📐 标准化 (Standardization)

从量纲统一到深度学习核心机制 — 理解 ML/DL/RL 中最重要的数据变换

📅 2026-04-28 🔖 ML 基础 · 深度学习 · 强化学习 🧑‍💻 Damon + Nemesis
💡 Damon 的直觉

"标准化就是把不同量纲的数据放到统一的一个框架里看待。"

这个直觉非常准确——标准化的本质就是消去量纲、建立统一坐标系，让来源不同、尺度各异的数据可以在同一把尺子下比较和处理。

1. 概念定义

标准化 (Standardization) 在统计学和机器学习中最常见的指代是 Z-score 标准化：

x' = (x − μ) / σ

其中 x 是原始数据，μ 是该特征的均值，σ 是标准差，x' 是标准化后的值，满足 均值为 0，标准差为 1。

直观理解：原始数据被平移（减去均值，消除偏移）再缩放（除以标准差，消除尺度差异），最终变成以 0 为中心、以 1 为单位的标准正态分布（假设原始数据近似正态）。

2. 标准化的三大作用

2.1 消除量纲

特征典型取值范围量纲
年龄0–100岁
收入0–10⁶元
身高140–200cm
点击率0–1无

直接把这样的数据扔进模型，收入（量级几万）会完全主导年龄（量级几十）——这不是因为收入更重要，纯粹是因为数字更大。标准化后，所有特征都在同一尺度上，模型的权重更新不再被量纲绑架。

2.2 加速优化收敛

在梯度下降中，未经标准化的特征会导致损失函数的等高线呈严重椭圆形，梯度方向不是直指最优点，而是来回震荡（zigzag），收敛缓慢。标准化后，等高线接近圆形，梯度直接指向最优方向。

2.3 满足模型假设

  • PCA：依赖方差最大化，量纲大的特征会主导主成分方向
  • SVM：基于距离，大尺度特征主导决策边界
  • KNN / K-Means：欧氏距离对尺度极度敏感
  • L1/L2 正则化：对特征尺度不公平

3. 标准化 vs 归一化

这是最常被混淆的两个概念。关键区别：

维度标准化 (Standardization)归一化 (Normalization)
公式x' = (x − μ) / σx' = (x − xmin) / (xmax − xmin)
输出范围无固定范围（通常 ~[-3, 3]）严格 [0, 1]
受异常值影响较小（均值和标准差相对稳健）极大（一个异常值直接压缩整个区间）
分布保持保持原始分布形状不保持，强行拉伸到 [0,1]
中心化✅ 以 0 为中心❌ 不以 0 为中心
典型场景PCA、SVM、逻辑回归、深度学习图像像素值、指标归一化

3.1 为什么深度学习偏好标准化？

  • 梯度流：以 0 为中心的数据让激活函数在对称区间工作，梯度流动更健康
  • BatchNorm 的设计前提：BatchNorm 内部做的就是标准化（减均值除标准差），整个 DL 的 normalization 家族都建立在标准化的思想之上
  • 异常值处理：Min-Max 会把所有值挤压到 [0,1]，一个异常值就能让 99% 的正常数据挤在 0~0.01 之间
Damon 的理解再深一层
"标准化把不同量纲放到统一框架里"，这个"框架"有两层含义：
表层：数值尺度统一（消除量纲）
深层：统计框架统一——把所有特征映射到同一个概率空间（均值 0、方差 1 的正态坐标系）
而归一化（Min-Max）只做到了"统一框架"的表层（都在 [0,1]），但没有改变分布的相对位置关系——它只是线性缩放，不中心化。

4. 在机器学习 (ML) 中的应用

4.1 距离/相似度敏感模型

算法为什么需要标准化
KNN距离计算依赖每个特征的权重。未标准化 = 大尺度特征主导
K-Means聚类中心的更新逻辑与 KNN 一致
SVM最大化间隔对尺度极度敏感
PCA方差最大化——量纲大的特征直接绑架主成分

4.2 基于梯度优化的模型

  • 逻辑回归 / 线性回归：收敛速度严重依赖特征尺度
  • 正则化（L1/L2）：惩罚项假设所有特征"同等重要"，但它看的是数值绝对值——不标准化就是在对正确的事施加错误的惩罚

4.3 特例：树模型免疫

树模型（决策树、随机森林、XGBoost、LightGBM）对特征尺度天然免疫——因为它们按特征值排序后做分割，只看相对顺序，不看绝对数值。这是一个常见的面试考点。

5. 在深度学习 (DL) 中的应用

这是标准化思想最精彩的延伸——从数据预处理发展为网络结构的一部分。

5.1 输入层标准化

训练前对所有输入特征做 Z-score 标准化是标配操作。不做的话，激活函数（sigmoid/tanh/ReLU）的工作区间会严重偏移。

5.2 Batch Normalization (BN) — 2015 里程碑

x̂ = (x − μbatch) / σbatch → y = γx̂ + β

核心思想：在网络内部也做标准化。不是对输入做一次就完，而是在每一层之后对 mini-batch 做标准化，再通过可学习的 γ（缩放）和 β（偏移）恢复网络的表达能力。

为什么 γ, β 是必须的？因为标准化到 N(0,1) 后如果直接喂给 sigmoid，大部分值会落在 sigmoid 的近似线性区——网络的非线性表达能力就废了。γ, β 让网络自己学习最佳的分布位置和尺度。

5.3 Layer Normalization (LN) — Transformer 标配

与 BN 的区别：BN 沿 batch 维度做标准化，LN 沿 feature 维度做标准化。

为什么 Transformer 用 LN 而不是 BN？

  • BN 在 NLP 中效果差：batch 内每个样本的序列长度不同，统计量不稳定
  • LN 对每个样本独立做，天然适配变长序列
  • LN 在推理时不需要依赖 batch 统计量

5.4 Normalization 家族全景

方法标准化维度典型场景
Batch Norm(N, H, W) — 沿 batch + 空间CNN 标配
Layer Norm(C, H, W) — 沿通道 + 空间Transformer / RNN
Instance Norm(H, W) — 单样本单通道风格迁移
Group Norm分组通道 + 空间小 batch 场景
RMS Norm省去中心化，只缩放LLaMA 等大模型

6. 在强化学习 (RL) 中的应用

RL 的训练极度不稳定，标准化在 RL 中甚至比在监督学习中更重要。

📊 状态标准化

维护观测值的 running mean/std，每个观测输入网络前先标准化。PPO/A3C 几乎必须做；OpenAI Five / AlphaStar 都使用。

🎯 奖励标准化

对奖励序列做 running 标准化，保持数值在合理范围。不做奖励标准化，RL 训练可能根本不收敛。

7. 总结：标准化的哲学

回到 Damon 的洞察——"把不同量纲的数据放到统一的一个框架里看待"。

1️⃣ 数据层

Z-score 标准化——把所有特征拉到同一统计坐标系，消除量纲偏见。

2️⃣ 模型层

Batch/Layer/Instance/Group Norm——把统一框架嵌入网络内部，让每一层都在稳定的分布上工作。

3️⃣ 训练层

RL 中的状态/奖励/优势标准化——在非平稳环境中持续维护统一框架，对抗分布漂移。

标准化的力量不在于"把数据变好看"，而在于建立一个公平的数值裁判体系——每个特征、每层网络、每步更新都在同一个标准下被衡量和调整。

归一化（Min-Max）只解决了"尺度统一"，而没有触及分布形态和中心化——这就是为什么深度学习偏爱标准化，而非简单 min-max 归一化。
