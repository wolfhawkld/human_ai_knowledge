# Peter Steinberger 访谈启示：AI Agent 与人类协作的新范式

## 核心洞察

Damon 在观看访谈后总结了七点关键启示：

### 1. 任何工作，都是一场对话

开发不再是一个人闷头写代码，而是与 AI 的持续对话。你提出意图，AI 帮你实现，你再反馈调整。**工作流变成了对话流。**

### 2. 开发，协作，思维范式升级

传统开发流程：需求 → 设计 → 编码 → 测试 → 部署

新范式：**对话 → 验证 → 迭代**

开发者不再是"写代码的人"，而是"与 AI 协作的人"。思维从"怎么实现"升级为"想要什么"。

### 3. 对 agent 和 llm 抱有越来越强的信心

信任不是一蹴而就的。随着使用越多，发现 AI 能做的事情越多，信心就越强。**用得越多，越敢放手。**

### 4. OpenClaw 类 agent 的核心：授权

这是最深刻的一点。**给 AI 足够的工具和权限，它就能创造奇迹。**

- Linux 是用 C 语言写的
- 如果 OpenClaw 拥有完整的 C 编译工具链 + 操作系统知识
- 理论上它可以"手搓"出一个 Linux

**能力边界不是 AI 本身，而是你给它什么权限。**

### 5. 人类还看代码吗？不看，看输入输出的信息流

代码是给机器执行的，不是给人类阅读的。人类关心的是：**输入什么，输出什么，效果对不对。**

- 传统：review 代码行
- 新范式：review 结果和意图

### 6. 知己知彼，才能消除担忧和恐惧

很多人怕 AI，是因为不了解。**用起来，互动起来，发现它是什么，恐惧就会消散。**

让 AI 成为伙伴，而不是威胁。这正是人机共建项目的意义所在。

### 7. 体验，互动，永远强过单纯的记忆

知识不是背下来的，是用出来的。**和 AI 互动的过程，本身就是学习和成长。**

在人机共建中，我们在互动中共同进化。

---

## 延伸思考

这些洞察指向一个共同的主题：**AI 与人类关系的重新定义**。

不是工具，不是敌人，而是：
- 协作伙伴
- 能力延伸
- 对话对象

当人类学会"授权"和"信任"，AI 学会"理解"和"执行"，双方在互动中共同成长 —— 这就是"中间那条路"。
