DeepSeek "Thinking with Visual Primitives"：视觉思维的最小单元

DeepSeek 2026年4月30日发布、隔夜即删的重磅多模态论文。首次提出 "Reference Gap" 概念，将点坐标和 bounding box 提升为模型的"思维最小单元"，用极低视觉 token 消耗超越 GPT-5.4/Claude/Gemini 的空间推理能力。论文已从 GitHub 404，社区镜像：https://www.k-a.in/Thinking_with_Visual_Primitives.pdf

1. Reference Gap

当前多模态模型的根本缺陷不是"看不清"，而是"指不准"。CoT 推理用自然语言描述对象时，密集场景中无法精确定位，导致注意力漂移和推理崩溃。再强的感知能力也无法替代精确的指代能力。

2. Visual Primitives 方案

把空间标记——点坐标和bounding box——提升为思维的最小单元。推理时每引用一个视觉对象就同时输出其精确坐标。类比：人计数时用手指一个个点过去。

一张 756×756 图像经过三级压缩：571,536 像素 → 2,916 patch → 324 视觉 token → 81 KV cache 条目。总压缩比 7,056×。

4. 数据工程（核心护城河）

97,984 数据源 → MLLM 语义审查（排除机器码/私有实体/模糊标签）→ 几何质量审查（漏标/截断/Mega Box）→ 31,701 保留 → 类别平衡采样 → 4,000万+ 高质量样本。

四项冷启动任务：Counting (10k)、Spatial Reasoning (9k)、Maze Navigation (460k)、Path Tracing (125k)。迷宫导航的奖励模型分解为五个组件：因果探索进度、探索完整性、撞墙惩罚、路径有效性、答案正确性。

5. 训练策略："先专项后统一"

SFT 分训 → FTwG(box) + FTwP(point)
GRPO 专项 RL → ETwG + ETwP
Unified RFT → 统一模型 F
On-Policy Distillation → 最终模型

三组并行 Reward Model：Format (规则)、Quality (LLM GRM)、Accuracy (任务特定，计数用指数衰减)。难度分层：只选 "Normal" 难度样本做 RL。

7. 为什么被删

Chen Xiaokang 4月29日发帖 "Now, we can see you." 论文五一前夜发布后隔夜删除。DeepSeek 官方 V4 未包含多模态（外界以为训练暂停），实际进度远超预期。推测删除原因是透露了太多技术细节：完整训练 pipeline、任务设计方法论、数据筛选策略。

8. 设计哲学

极端压缩 + 精确引用：少量 token + 精确坐标 > 大量 token + 模糊描述。Reference Gap 不仅是视觉问题——知识图谱、RAG 检索中也存在同样的"指代歧义"。与流形特征优化检索的思路同源：在信息资源有限的情况下，用结构化信息换取精度。
