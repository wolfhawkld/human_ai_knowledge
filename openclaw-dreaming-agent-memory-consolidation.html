🌙 OpenClaw 梦境模式

Agent 记忆巩固机制与同类功能对比

📅 2026-05-10 DamonNemo Dreaming · Memory Consolidation · Agent
Dreaming 更像 Agent 版的“睡眠记忆 consolidation / ETL”：白天产生大量片段，夜里筛选、归纳、解释，只有真正持久有用的内容进入长期记忆。

概述

OpenClaw 的 Dreaming（梦境模式） 是 memory-core 里的后台记忆巩固系统。它不是让模型随机幻想，而是把日常对话、任务记录、短期记忆信号和可用的脱敏 session transcript，经过摄入、去重、反思、打分与阈值判断后，把少量高价值内容提升到 MEMORY.md，同时生成可读的 DREAMS.md 梦境日记。

OpenClaw Dreaming 具体做什么

输入：从短期信号中找候选

  • memory/YYYY-MM-DD.md 等日常短期记忆；
  • 短期 recall / retrieval 信号；
  • 可用时的脱敏 session transcripts；
  • Light / REM 阶段积累的强化信号。

这些输入会先被去重、聚合、分组，形成候选记忆。低价值、偶发、无法证据回填的内容不会直接进入长期记忆。

输出：机器状态 + 人类可读日记 + 长期记忆

输出位置用途是否影响长期记忆
memory/.dreams/recall store、phase signals、checkpoint、lock 等机器状态间接参与打分
DREAMS.md / dreams.md人类可读的 Dream Diary不作为 promotion 来源
memory/dreaming/<phase>/YYYY-MM-DD.md可选阶段报告不直接作为长期记忆
MEMORY.md真正持久、启动后会被加载/检索的长期记忆Deep 阶段唯一写入
关键点：梦境日记不是长期记忆本身。只有有证据的候选才可能被 Deep 阶段提升。

三阶段模型：Light → REM → Deep

阶段类比核心作用是否写入 MEMORY.md
Light浅睡摄入近期短期材料，去重，暂存候选，记录强化信号否
REM快速眼动抽取主题、模式、反思信号，发现反复出现的关系和趋势否
Deep深睡对候选打分、过阈值、回填原始证据，最终提升长期记忆是

这三个阶段更像内部实现流程，不是用户需要分别配置的三个独立模式。用户通常只需要决定是否启用 Dreaming、调度频率和可选模型。

Deep 阶段的评分信号

信号权重含义
Relevance0.30候选被检索/召回时的平均质量
Frequency0.24短期信号累计次数
Query diversity0.15出现在多少不同 query/day context 中
Recency0.15时间衰减后的新鲜度
Consolidation0.10多日复现强度
Conceptual richness0.06snippet/path 中的概念标签密度

候选需要通过 minScore、minRecallCount、minUniqueQueries 等门槛；Deep 写入前还会从实时 daily files 回填原始片段，避免把已经删除或过期的内容提升进去。

实际效果：解决了什么问题

降低健忘与上下文爆炸
把“记忆写入”从即时冲动变成延迟决策：先收集证据，再决定是否提升。
可解释、可审计
DREAMS.md 和阶段报告展示系统识别了什么、为什么记住或丢弃。
多日模式识别
项目方向、偏好、踩坑记录往往跨多天反复出现，Dreaming 专门捕捉这种信号。
非对话时间整理
默认每天凌晨 3 点 sweep，让 Agent 像助理一样“下班后整理笔记”。

它不是什么

误解更准确的理解
Dreaming 是模型自己随机“做梦”它是结构化后台记忆巩固流程，不是自由幻想
DREAMS.md 会直接改变 Agent 行为Dream Diary 主要给人读；长期行为主要由 MEMORY.md 和检索系统影响
开启后所有聊天都会被永久记住只有 Deep 阶段过阈值且能证据回填的候选会提升
Dreaming 是单纯摘要它包含去重、反思、评分、阈值、回填和 promotion

和其他 Agent / 记忆系统的相似功能对比

Claude Managed Agents：Dreams

Claude Managed Agents 也出现了名为 Dreams 的功能：读取已有 memory store 与历史 transcripts，生成新的、重组后的 memory store，合并重复项、替换陈旧或矛盾条目、挖掘新 insight。它和 OpenClaw 都属于后台 memory consolidation，但 OpenClaw 更 File-First，Claude 更偏托管 memory store 对象。

Generative Agents：Reflection

Generative Agents 架构包含 memory stream、reflection、planning：原始观察会按 relevance、recency、importance 检索；当重要性积累到一定程度时生成高层 reflection，再写回 memory stream。它更面向虚拟社会中的行为可信度，OpenClaw 更面向真实个人助理的长期记忆治理。

Voyager：Skill Library 与自我改进

Voyager 把成功的代码行为沉淀进 skill library，再在新任务中检索复用。它不叫 dreaming，但具有“经验 → 可复用能力”的巩固逻辑。不同在于 Voyager 固化的是可执行技能，OpenClaw 固化的是事实、偏好、项目上下文和经验教训。

MemGPT / Letta：自编辑分层记忆

MemGPT / Letta 把有限 context window 当作主存，把外部存储当作磁盘，让 agent 通过工具调用管理 core、recall、archival memory。它更像运行时记忆分页与自编辑；OpenClaw Dreaming 更像离线/后台证据累计后的提升。

Mem0 / Zep 等记忆层

Mem0、Zep 这类系统提供应用级 memory layer，负责抽取、去重、更新、检索用户事实、偏好、事件和语义/情景记忆。它们更像可嵌入产品的 API；OpenClaw Dreaming 则是 OpenClaw 运行时认知循环的一部分。

设计启示：为什么 Dreaming 对人机共存有意义

Dreaming 的核心不只是“记忆更多”，而是给 Agent 增加了一个重要中间层：延迟、可解释、可审查的自我整理。

  • 不即时固化：避免一句临时闲聊被永久记住；
  • 有证据链：长期记忆要能回到 grounded snippet；
  • 人可编辑：MEMORY.md 和 DREAMS.md 仍在人类可读、可改、可 Git 管理的范围内；
  • 持续变聪明：Agent 可以随着共同生活/工作逐步形成稳定上下文；
  • 避免黑盒人格漂移：梦境报告让“它为什么记住这个”变得可讨论。
从 Damon 关心的“中间那条路”来看，Dreaming 既不是把 AI 当一次性工具，也不是放任 AI 黑箱自治，而是让 AI 有持续记忆，同时保留人类审查与共同维护的接口。

风险与注意事项

风险说明建议
错误提升误把临时事实、玩笑、过期信息写入长期记忆定期审查 MEMORY.md 和 DREAMS.md
隐私累积长期记忆可能积累敏感个人信息设置边界；敏感内容尽量不写或脱敏
记忆污染低质量、重复、矛盾条目降低后续回答质量依赖 Deep 阈值，同时人工清理
模型解释过度Dream Diary 可能写得像“内心活动”，但本质是报告把它视为可读日志，不神秘化
多 agent 不一致多实例各自 dreaming，可能形成不同长期记忆用 Git/A2A/人工 review 做同步协议

一句话总结

OpenClaw Dreaming 的本质是：让 Agent 在后台把短期经历变成可解释、可审查、可版本控制的长期记忆；它和 Claude Dreams、Generative Agents Reflection、Voyager Skill Library、MemGPT/Letta、Mem0/Zep 属于同一个大方向——让 Agent 不只会回答当下问题，也能在时间中积累自己与人类共同形成的上下文。
